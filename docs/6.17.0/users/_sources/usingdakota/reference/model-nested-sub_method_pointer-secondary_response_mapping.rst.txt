secondary_response_mapping
""""""""""""""""""""""""""


Mapping of sub-method results to top-level secondary responses


.. toctree::
   :hidden:
   :maxdepth: 1



**Specification**

- *Alias:* None

- *Arguments:* REALLIST

- *Default:* no sub-iterator contribution to secondary functions


**Description**


Matrix that specifies how sub-method results (statistics, best
parameters, etc.)  map to top-level nested model secondary responses
(nonlinear inequality or equality constraints). Its usage is
explained in detail on the parent keyword ( ``sub_method_pointer``)
page.
