primary_response_mapping
""""""""""""""""""""""""


Mapping of sub-method results to top-level primary responses


.. toctree::
   :hidden:
   :maxdepth: 1



**Specification**

- *Alias:* None

- *Arguments:* REALLIST

- *Default:* no sub-iterator contribution to primary functions


**Description**


Matrix that specifies how sub-method results (statistics, best
parameters, etc.)  map to top-level nested model primary responses
(objectives, calibration terms, or response functions). Its usage is
explained in detail on the parent keyword ( ``sub_method_pointer``)
page.
