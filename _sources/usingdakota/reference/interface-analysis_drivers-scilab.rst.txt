.. _interface-analysis_drivers-scilab:

""""""
scilab
""""""


Run Scilab through a direct interface - requires special Dakota build


.. toctree::
   :hidden:
   :maxdepth: 1



**Specification**

- *Alias:* None

- *Arguments:* None


**Description**


Dakota supports :ref:`a library-linked interface to Scilab <advint:scilab>`, but it must be
explicitly enabled when compiling Dakota from source.  Contact the
Dakota users mailing list for assistance building and using Dakota
with these interfaces.

The :ref:`interface-analysis_drivers<interface-analysis_drivers>` specifies a Scilab file which
implements the parameter to response mapping.



**Examples**


See ``dakota/share/dakota/examples/users/Scilab/linked``
